# 내용 기반 레이아웃 라우팅

공통 양식은 고정하되 본문은 원문의 관계, 읽는 순서, 증거 밀도에 맞춰 구성한다. 카드 수나 문단 수만으로 레이아웃을 선택하지 않는다.

## 선택 절차

1. 원문의 핵심 명사를 노드로, 동사를 관계로 추출한다.
2. 주된 관계를 하나 고른다: 흐름, 분해, 상호작용, 통제, 물리 배치, 시간, 반복, 산출물, 비교, 수치.
3. 독자가 따라야 할 읽기 순서를 정한다: 좌→우, 상→하, 중심→외곽, 시간축, 반복 순환, 비교 축.
4. 아래 표에서 주 레이아웃을 선택하고 보조 요소를 최대 두 종류만 붙인다.
5. 두 개의 와이어프레임 후보를 만들고 의미 적합성, 스캔 경로, 정보 밀도, 인접 장표와의 중복을 비교한다.
6. 렌더 후 빈 공간과 과밀 영역이 동시에 나타나면 글자 크기보다 레이아웃 선택을 먼저 재검토한다.

## 관계별 주 레이아웃

| 관계 유형 | 주 레이아웃 | 적합한 내용 | 피해야 할 대체 |
|---|---|---|---|
| 전체 흐름·목표 구조 | End-to-End 아키텍처 | Agent→Gateway→Server→API, 목표 구조 | 의미 없는 동일 카드 나열 |
| 구성요소 분해 | 계층형 스택·컴포넌트 맵 | Microservice, 책임, 배포 경계, 공통 Core | 시간 순서 화살표 |
| 프로토콜·호출 | 스윔레인·시퀀스 | Client/Gateway/Server 요청·응답, 오류 규약 | 설명 카드만 사용 |
| 보안·운영 통제 | 중앙 허브·방사형 통제 | Identity, 개인정보, 감사, HA | 선형 로드맵 |
| 물리 구축·배포 | 영역형 토폴로지 | WEB/K8s/DB, 이중화, NetworkPolicy | 추상 카드 그리드 |
| 단계·확산 | 타임라인·계단형 로드맵 | PoC→시범→확산, 수량, 전환 | 순환 허브 |
| 반복 처리 | 파이프라인·순환 루프 | 조사→정의→Adapter→시험→배포 | 정적인 조직도 |
| 산출물·완료 | 라이프사이클 + 완료 Gate | 설계서, 검토, 승인, 인계 | 단순 아키텍처 흐름 |
| 비교·선정 | 2축 매트릭스·평가기준 표 | 우선순위, 대안, 적합성 평가 | 순차 화살표 |
| 현황·성과 수치 | KPI 대시보드·차트 조합 | 목표치, 현황, 추이, 분포 | 장식용 도형만 사용 |
| 전후 변화 | Before/After·브리지 | 현행 문제, 개선 구조, 기대 효과 | 서로 무관한 4카드 |
| 계층·책임 | 트리·계단형 책임 구조 | 조직, 역할, 승인선, 상하위 관계 | 방사형 허브 |

## 후보 평가

| 기준 | 질문 |
|---|---|
| 의미 적합성 | 도형의 위치와 연결이 원문의 관계를 설명하는가? |
| 읽기 경로 | 제목에서 결론까지 시선 이동이 자연스러운가? |
| 증거 가시성 | 요구사항, 수량, 역할, 산출물이 묻히지 않는가? |
| 밀도 균형 | 큰 빈 공간 옆에 작은 글자가 몰리지 않는가? |
| 편집성 | 주요 텍스트와 시각 요소를 PowerPoint에서 바로 선택할 수 있는가? |
| 묶음 다양성 | 인접 장표와 다른 관계인데 같은 골격을 반복하지 않는가? |

## 배치 규칙

- 한 장표에는 하나의 주 시각 구조만 둔다.
- 보조 카드는 주 구조를 설명할 때만 사용한다.
- 같은 레이아웃을 연속 사용하려면 원문의 관계와 읽기 경로도 같아야 한다.
- 3장 이상이 모두 같은 골격이면 각 장의 관계 분류를 다시 검토한다.
- 6장 이상의 배치에서는 내용이 허용하는 범위에서 세 종류 이상의 레이아웃 계열을 사용한다.
- 색상만 바꾼 구조 반복은 다양성으로 보지 않는다.
- 마지막 접촉면 검사에서는 제목을 가려도 각 장표의 본문 골격이 구분되어야 한다.

## 예시: 3.1 묶음

- `3.1.1`: 목표 아키텍처
- `3.1.2`: 계층형 Microservice 맵
- `3.1.3`: 프로토콜 스윔레인
- `3.1.4`: 보안·운영 통제 허브
- `3.1.5`: Kubernetes 배포 토폴로지
- `3.1.6`: 전환·확산 로드맵 + 반복 파이프라인
- `3.1.7`: 산출물 라이프사이클 + 완료 Gate

공통 크롬, Pretendard, 팔레트, 선, 모서리, 메시지 밴드는 통일하되 본문 좌표 골격은 관계에 따라 달라져야 한다.
