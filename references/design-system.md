# 제안서 디자인 시스템

## 목표

`2-3-1.pptx`와 사용자가 제공한 통합 제안서의 시각 체계를 재현한다. 공통 크롬과 톤은 고정하지만 본문은 내용에 맞는 여러 레이아웃을 사용한다. 편집성과 MS PowerPoint 호환성을 유지하면서 참조본의 정보 밀도와 계층을 재현한다.

## 대지와 안전영역

- 대지: 7.5 × 10.8333in, 세로, 흰색.
- 상단 템플릿: y=0.00~1.25in.
- 본문 안전영역: x=0.42~7.08in, y=1.30~9.78in.
- 하단 템플릿: y=10.02~10.50in.
- 좌우 기본 여백: 0.42~0.55in.

## 공통 크롬

- 상·하단 네이비 띠 또는 제안서 원본의 동일 요소를 유지한다.
- 제목 왼쪽에 얇은 파란 세로 강조선을 둔다.
- 절 번호는 파란색 둥근 캡슐에 흰색 12pt 이상으로 표시한다.
- 제목은 승인 예시의 계층을 유지하기 위해 Pretendard 20~22pt 굵게 사용하고, 장 상단 section title은 14pt, Chapter code는 20pt로 고정한다.
- 부제는 Pretendard 11~12pt로 제목 바로 아래에 배치한다.
- 푸터와 페이지 번호는 원본 제안서 위치를 유지하며 10~12pt를 사용한다.

## 기준 팔레트

- 네이비: `0B2A53` 또는 참조본에서 측정한 가장 가까운 값.
- 핵심 블루: `2F66F5`.
- 보조 퍼플: `7C3AED`.
- 보조 틸: `0F8C7A`.
- 보조 앰버: `F5A000`.
- 본문: `1F2937`.
- 보조 본문: `64748B`.
- 선: `D6E3F3`.
- 패널: `F7FAFE`.
- 블루 연색: `EAF2FF`.
- 퍼플 연색: `F1EBFF`.
- 틸 연색: `E9F8F5`.
- 앰버 연색: `FFF5DF`.

페이지마다 네이비+블루를 기본으로 두고, 의미상 필요한 두 개의 보조색만 사용한다.

## 레이아웃 문법

- 공통 크롬 다음에 내용의 관계를 가장 잘 설명하는 주 시각구조를 하나 배치한다.
- 아키텍처, 계층형 스택, 스윔레인, 통제 허브, 배포 토폴로지, 로드맵, 파이프라인, 산출물 Gate를 구분하여 사용한다.
- 큰 연한 패널, 얇은 테두리, 의미색, 아이콘 배지, 컴팩트 라벨은 모든 유형에서 공통으로 사용한다.
- 카드 제목과 본문은 하나의 텍스트 포함 도형 안에 둔다. 카드는 주 구조를 설명할 때만 사용한다.
- 상세 선택 규칙은 `layout-routing.md`를 따른다.

### 핵심 메시지

- 왼쪽의 작은 파란 라벨과 오른쪽의 진한 네이비 메시지 밴드를 사용한다.
- 메시지는 한 문장으로 압축하고 12pt 굵게 표시한다.

## 타이포그래피

- 모든 글꼴: Pretendard.
- 제목: 20~22pt.
- 영역 제목: 16~18pt.
- 카드 제목: 12~14pt 굵게.
- 본문: 12pt 권장, 절대 10pt 미만 금지.
- 푸터·페이지 번호: 10~12pt.
- 모든 크기는 자연수.
- 자동 축소를 사용하지 않는다. 넘치면 문장을 줄이거나 도형을 키운다.

## 밀도 목표

- 장표당 약 30~60개의 편집 가능한 개체를 목표로 한다.
- 아이콘 또는 일러스트 4~10개를 사용한다.
- 주 시각구조와 2~6개의 보조 설명 단위를 사용한다.
- 빈 공간을 장식으로 억지로 채우지 말고, 원문의 역할·통제·효과·산출물 정보를 활용한다.
- 12pt 기준으로 들어가지 않는 내용은 중복 접속어와 반복 설명을 먼저 제거한다.

## 금지 사항

- 텍스트가 없는 배경 카드 위에 같은 카드의 제목·본문을 별도 텍스트박스로 겹쳐 놓지 않는다.
- 텍스트 포함 개체에 그림자, Glow, Bevel, 3D, Reflection을 적용하지 않는다.
- 전체 본문을 PNG/JPG/SVG 한 장으로 변환하지 않는다.
- 참조본의 9pt 텍스트, 오버플로, 겹침, OOXML 오류를 복제하지 않는다.
- OpenAI 로고, 프롬프트, 내부 지시문을 넣지 않는다.
