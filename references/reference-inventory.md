# 참조 장표 인벤토리

## 공통 디자인 기준

### `assets/reference-decks/2-3-1.pptx`

- 사용 목적: 공통 크롬, 아키텍처형 상단 흐름, 역할 설명, 핵심 메시지 밴드, 여백과 정보 밀도.
- 따라야 할 요소: 세로 강조선, 절 번호 캡슐, 큰 연색 패널, 아이콘 배지, 색상별 카드, 하단 메시지.
- 복제하지 말아야 할 결함: 9pt 본문, 텍스트 넘침, 검증 오류.

## 레이아웃 보조 템플릿

### `assets/reference-decks/2-3-6.pptx`

- 사용 목적: 관측·모니터링·데이터 파이프라인처럼 순서가 중요한 기술 흐름.
- 강점: 상단 파이프라인과 하단 2×2 카드의 균형, 아이콘 중심 설명, 운영효과 밴드.
- 적용 상황: 모니터링, 연계 파이프라인, 데이터 처리, 단계형 운영구조.

### `assets/reference-decks/2-4-7.pptx`

- 사용 목적: 산출물 라이프사이클과 단계별 제출·인수인계 구조.
- 강점: 5단계 상단 흐름, 4+1 카드, 색상별 단계 구분, 핵심 메시지.
- 적용 상황: 산출물, 로드맵, 단계별 전환, 완료·인계기준.
- 복제하지 말아야 할 결함: 9pt 문구와 일부 높이 부족.

## 사용 절차

1. 사용자가 통합 제안서 덱을 제공하면 section II 이후의 대표 장표를 여러 장 렌더링하여 실제 레이아웃 라이브러리를 만든다.
2. `2-3-1`을 공통 디자인 및 아키텍처형 기준으로 측정한다.
3. 작성 대상의 관계 유형을 `layout-routing.md`로 판단한다.
4. 파이프라인에는 `2-3-6`, 산출물에는 `2-4-7`을 사용하고 나머지는 관계에 맞는 구조를 새로 조합한다.
5. 여러 페이지면 먼저 레이아웃 배정표를 작성하고 인접 페이지의 골격 반복을 피한다.
6. 색상·크롬·밀도는 항상 같은 제안서 계열로 되돌린다.
7. 참조 PPTX 자체를 결과물로 복사하거나 수정하지 않는다. PptxGenJS로 구조를 재구성한다.

## 검증된 reference-led KISTI 예시

- 위치: `assets/examples/reference-led-kisti/`
- 구성: 편집형 단일 장표 PPTX 8개와 native preview 8개. 재현 가능한 소스는 `assets/pptxgenjs-kit/`의 `proposal-kit.mjs`·`page-template.mjs`·`reference-layouts.mjs`와 `reference-media/`에 단일화한다.
- 용도: 21·27·42·52 장표 계보를 한 디자인 시스템으로 통합하면서 계층형·기본형·파이프라인·산출물형·Stage Gate·장애대응·품질허브·화면증거형으로 확장하는 예시.
- 사용 전 [reference-led-patterns.md](reference-led-patterns.md)를 읽고, 예시의 문구가 아니라 관계 유형·읽기 경로·공통 크롬을 재사용한다.
- 새 작업 폴더는 `scripts/scaffold-page.ps1`로 만들고, 복사된 `reference-layouts.mjs`를 좌표·objectName 패턴 라이브러리로 사용한다.
