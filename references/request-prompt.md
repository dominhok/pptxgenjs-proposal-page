# 재사용 요청 프롬프트

다음 요청을 작업 계약으로 해석한다.

> 위 작성 대상 내용을 포함하여 발표자료용 인포그래픽 PPTX를 생성한다. 새로운 디자인 계열을 만들지 않고 참조 제안서의 공통 크롬·톤앤매너·정보밀도를 유지한다. 다만 `2-3-1.pptx` 한 장의 내부 레이아웃을 모든 절에 복제하지 말고, 참조 덱의 다양한 장표와 작성 내용의 관계 유형을 분석하여 절별로 적합한 레이아웃을 선택한다.

필수 조건:

- PptxGenJS로 생성하고 OfficeCLI로 후처리·검증·렌더링한다.
- A4 세로 7.5 × 10.8333in, 흰색 배경, Pretendard, 한국어를 사용한다.
- 제목·본문을 배경 카드와 분리된 별도 텍스트박스로 겹치지 않고 텍스트 포함 도형으로 만든다.
- 본문 12pt를 권장하며 10pt 미만을 사용하지 않는다. 모든 글자 크기는 자연수다.
- 텍스트 포함 개체에 그림자·Glow·Bevel·3D·Reflection을 적용하지 않는다.
- 아키텍처·계층형 맵·스윔레인·통제 허브·배포 토폴로지·로드맵·산출물 Gate 중 내용에 맞는 주 구조를 선택하여 참조본 수준의 정보 밀도를 유지한다.
- 여러 절을 함께 생성하면 인접 장표가 동일한 본문 골격으로 보이지 않도록 하고, 3장 이상이면 최소 3개 레이아웃 유형을 사용한다.
- 주요 텍스트는 모두 편집 가능한 PowerPoint 텍스트로 유지하고 전체 인포그래픽을 래스터화하지 않는다.
- 주요 도형·텍스트·화살표·아이콘을 PowerPoint에서 각각 바로 선택·수정할 수 있게 유지한다.
- 개체에 `Flow__*`, `Cards__*`, `Message__*`, `Support__*` 형식의 의미 있는 이름을 부여해 PowerPoint 선택 창에서 쉽게 찾을 수 있게 한다.
- LibreOffice, PowerPoint COM, python-pptx, 임의 OOXML 편집을 사용하지 않는다. 단, 번들된 스크립트의 PptxGenJS 미사용 노트 파트 정규화만 허용한다.
- MS PowerPoint 2021 이상을 대상으로 OOXML 검증, 넘침 검사, 렌더링 검수를 통과한다.
- OpenAI 로고, 프롬프트, 내부 지시사항을 슬라이드에 넣지 않는다.
- 파일명은 절 번호의 점을 하이픈으로 바꾼다. 예: `3.2.4` → `3-2-4.pptx`.

최종 검수에서는 참조본과 나란히 비교하여 정보 밀도, 색상, 크롬, 여백감이 같은 디자인 계열인지 확인하고, 묶음 접촉판에서 내용이 다른 장표들이 같은 골격으로 반복되지 않는지 확인한다.
